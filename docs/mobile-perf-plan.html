Why your app feels like 2020 — and how to make it feel like 2026.

Three independent audits of The Dad Center mobile app converged on the same root causes: a prefetcher that never hits the cache, all five tabs mounting at once, eight spring animations stacked under eight BlurViews, and an auth waterfall blocking the first paint. Once data lands the app is fast — that's the tell. The pain is structural, not infrastructural. Fixable in a day for ~80% of the win.

Diagnosis

Symptoms map cleanly to causes.

Each thing you experience as "slow" or "stuck" has a specific structural cause. None of these are mysterious.

Slow first load

"Tabs and screens take forever the first time."

caused by →
Auth waterfall (profile → family) blocks route protection. Prefetcher writes to one cache key while the UI reads from another, so 5+ queries fire cold on every dashboard mount.

Stuck tab switches

"Sometimes screens hang or animations stutter."

caused by →
All 5 tabs and 21 More sub-screens mount on first render. Dashboard's LinearTransition.springify() re-springs all 8 cards on any data change. Stacked BlurViews + spring animations saturate the GPU.

Wrong page from notification

"I tap a notification and end up somewhere else."

caused by →
Notification listener uses a hardcoded 500ms delay, racing the auth redirect. If auth resolves first, it clobbers the deep-link push. Unknown URLs silently fall back to /(tabs).

Generally janky feel

"Even when things load it doesn't feel smooth."

caused by →
18 infinite Reanimated worklets running at all times (6 particles × 3 animations each). Old React Native architecture means every navigation crosses the bridge. TabHeader re-renders on every route change.

The 10 root causes

Ranked by priority. Three audits, same conclusions, all verified with file:line references in the actual code.

P0

Prefetch keys don't match dashboard query keys

The auth provider prefetches with ['tasks-due'] but the dashboard queries with ['tasks-due', family.id]. Cache miss every time. Persistent 7-day cache is wasted bytes for these keys. This is the single biggest win.

components/providers/AuthProvider.tsx:171–200
Catastrophic

All 5 tabs + 21 sub-screens mount eagerly

Commit 57698eb reverted both lazy:true and freezeOnBlur:true because nested Stacks broke. Hypothesis: only freezeOnBlur was the culprit (it freezes Stack state mid-navigation). Re-enable lazy alone.

app/(tabs)/_layout.tsx:117–179

Dashboard: 8 staggered springs + 8 BlurViews + LinearTransition

Eight CardEntrance wrappers fire FadeInDown.springify() with 40–360ms delays. All wrapped in an Animated.View layout={LinearTransition.springify()}, so any single card refresh re-springs all 8 siblings. Each card is a GlassCard = a separate iOS BlurView.

AuthProvider waterfall on boot

Sequential awaits: profile fetch → then family fetch. Route protection blocks on both. Adds an entire round-trip (200–400ms on cellular) before the UI can render anything.

Notification routing race condition

500ms hardcoded delay races AuthProvider's redirect. Auth wins → notification deep-link is overwritten. Unknown URLs silently default to /(tabs) with no logging. This is the "wrong page" bug.

Dashboard query explosion

5 hooks fire 5 separate queries on mount. useBacklogCount, useArticle have no staleTime set → refetch on every mount. Subcomponent cards each fire their own queries. Nothing is batched.

No New Architecture (Fabric/TurboModules)

Running on the old React Native architecture. Every navigation = bridge calls = main thread blocks. Reanimated 4.2.1 is Fabric-ready and would unlock sustained 60fps. Risk: native module audit required.

Industry Patterns

How modern apps solve this.

Every problem you have is a solved problem in 2026. Linear, Notion, Discord, Shopify, Robinhood — they all use the same core patterns.

Problem 2026 best-in-class pattern Apps that do this
Cold load feels dead Skeleton screens matching final layout — never spinners. Linear, Notion, Airbnb
Tab-switch lag Lazy mount + view-state preservation. Mount on first visit, keep mounted after. Instagram, Discord RN
Stale-while-revalidate feel React Query staleTime + persister with matching keys; Suspense boundaries. Linear web, GitHub Mobile
Animation cost on mount One container transition, not N staggered springs. Skip animation when cache is warm. Robinhood, Airbnb
Heavy blur One BlurView behind opaque cards. Apple HIG explicitly says never stack BlurViews. iOS Control Center, Apple Music
Long lists FlashList (Shopify) for virtualization; memoized filters keyed on stable inputs. Shopify, Instagram Reels
Auth waterfall Single RPC returns { profile, family, babies } in one round-trip. Linear, Superhuman, Cash App
Notification race Deep-link intent queue, drained once auth resolves. Slack, Gmail
Native perf ceiling Hermes + New Architecture (Fabric + TurboModules) so Reanimated/Gesture Handler bypass the bridge. Discord RN, Shopify, Expo
Predictive prefetch Pre-warm next route on touch-down/intent, not just at login. Linear, Arc

Fix Plan

Three phases. Most of the win in the first day.

Phase 1 fixes the cache, lazy-loads tabs, and adds skeletons — 80% of the perceived improvement, very low risk. Phase 2 closes the gap. Phase 3 is the big native-level overhaul (highest ceiling, real risk).

PHASE 01

Quick wins

~1 day ~80% perceived Low risk
  • Match prefetch keys to query keys Move prefetch into fetchProfile after family.id is known. Use exact same keys as useDashboardData. Instantly turns the 7-day persister into a real cache. AuthProvider.tsx:171–200 · use-dashboard.ts:25,31
  • Re-enable lazy:true on tabs Without freezeOnBlur (that was the actual culprit). Only Home mounts on first render. Tasks/Briefing/Tracker/More mount on first visit. app/(tabs)/_layout.tsx:117–179
  • Skeleton screens Replace ActivityIndicator with skeleton matching final card outlines. Same on tasks + briefing tabs. app/(tabs)/index.tsx · tasks/index.tsx · briefing/index.tsx
  • Cut dashboard animation cost Remove LinearTransition.springify() wrapper. Compress stagger 40–360ms → 0–120ms. Skip entrance animation when cache is warm. app/(tabs)/index.tsx:103–190
PHASE 02

Meaningful gains

1–2 days +15% perceived Moderate
  • Memoize TabHeader + BabySwitcher Move useSegments() into a tiny inner child. Wrap with React.memo. Stops per-route header re-render cascade. app/(tabs)/_layout.tsx:51–102
  • Collapse profile + family into one Supabase RPC New get_initial_app_state(user_id) returns { profile, family, babies } in one round-trip. Saves 200–400ms cold start. AuthProvider.tsx:63–106
  • Add staleTime to all hooks missing it Default 30s–5min. Stops refetch-on-every-mount when bouncing between tabs. use-triage.ts:30 · use-content.ts:18 · sweep all hooks/
  • Fix notification race Replace 500ms timeout with intent queue. Drain after auth resolves. Log unknown URLs to Sentry. NotificationListener.tsx:50–92
  • Reduce particle count 6 → 3, or replace with static SVG. Memoize particle colors. Frees 9–18 worklets. FloatingParticles.tsx:59–130
PHASE 03

Native overhaul

2–3 days Highest ceiling Real risk
  • Enable New Architecture Add newArchEnabled: true. Audit every native module first (BlurView, RevenueCat, Sentry, etc.). Gate behind 1-week TestFlight alpha. app.config.ts
  • Kill stacked BlurViews One BlurView in AppBackground. GlassCard becomes opaque rounded view with semi-transparent fill. Apple HIG-compliant. GlassCard.tsx · AppBackground.tsx
  • Memoize task list filters One useMemo for the chained filters. Precompute isToday() per task, store as flag. app/(tabs)/tasks/index.tsx:129–533
  • FlashList for tasks Swap FlatList → @shopify/flash-list. Required for >50 task lists. app/(tabs)/tasks/index.tsx
  • Suspense boundaries per tab Wrap each tab body with <Suspense fallback={Skeleton}>. Convert primary queries to useSuspenseQuery. app/(tabs)/*/index.tsx

Guardrails

Things to specifically not do.

Each of these is a tempting shortcut that has a known failure mode. Don't take them.

×
Don't re-attempt freezeOnBlur:true. That's the stuck-route / wrong-page bug from commit 57698eb. Only re-enable lazy:true.
×
Don't enable New Architecture without a native module audit. One non-Fabric module = broken build for everyone. Audit BlurView, RevenueCat, Sentry, gesture-handler, async-storage first.
×
Don't introduce new abstractions. No PerformanceProvider, no HOCs, no context-in-context. Fix in place.
×
Don't replace React Query. The persister + staleTime model is correct. The bug is the key mismatch — fix that, the model works as intended.
×
Don't remove the AsyncStorage persister. Once Phase 1.1 lands, it becomes a real 7-day cache asset, not a wasted byte field.
×
Don't ship New Arch to prod without a week in TestFlight alpha with Sentry crash monitoring on. The one-time risk is worth a careful rollout.
×
Don't animate during cold-mount if cache was warm. Animations communicate freshness. Warm cache means "already here." Skipping the animation makes it feel instant.
