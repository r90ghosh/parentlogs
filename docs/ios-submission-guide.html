Build & Submit

Build the production binary and submit it to App Store Connect.

Run production build Command
cd apps/mobile
eas build --platform ios --profile production
This takes 15-30 minutes. EAS will:
  • Build the iOS binary in the cloud
  • Auto-increment the build number
  • Sign with your distribution certificate
  • Upload source maps to Sentry (if configured)
You can monitor the build at expo.dev > Your Project > Builds.
Submit to App Store Connect Command
After build completes: 
eas submit --platform ios --profile production
Or combine build + submit in one command: 
eas build --platform ios --profile production --auto-submit
EAS will upload the .ipa to App Store Connect. This takes ~5 minutes.
Verify build appears in App Store Connect Apple
Go to App Store Connect > Your App > TestFlight. The new build should appear within 5-15 minutes after submission. It will show as "Processing" initially, then "Ready to Test".

If build doesn't appear: Check your email for any "Missing Compliance" alerts. You may need to answer the export compliance question (select "No" for encryption since usesNonExemptEncryption: false is already set).

TestFlight Testing

Test on real devices via TestFlight before submitting for review.

Install via TestFlight on your device Manual
  1. Open TestFlight on your iPhone
  2. The new build should appear automatically (you're the developer)
  3. Tap "Install" on the latest build
Test: Guest browsing flow Manual
  • Open app fresh (not logged in)
  • Verify landing page loads with hero, features, pricing
  • Tap "Browse as Guest" → verify 3 tabs (Home, Explore, Pricing)
  • Tap each content card in Explore → verify screens load
  • Tap "Get Started" CTA → verify it goes to signup
Test: Signup & onboarding Manual
  • Sign up with email + password
  • Sign up with Apple (test Apple Sign In works)
  • Complete onboarding: role → family (due date) → invite partner → ready
  • Verify you land on the main dashboard
Test: Core features Manual
  • Dashboard: Cards load, greeting shows, quick actions work
  • Tasks: View tasks, complete a task, create a task
  • Briefing: Current week loads, navigate to other weeks
  • Tracker: Log a feed/diaper/sleep, view history
  • More: All menu items navigate correctly
  • Budget: Templates load, add item to budget
  • Checklists: View checklists, toggle items
Test: Subscriptions (sandbox) Critical
  1. Go to Settings > Upgrade (or the paywall)
  2. Tap a subscription plan
  3. Complete the sandbox purchase (use your Sandbox Apple ID)
  4. Verify subscription tier updates in the app
  5. Verify premium features unlock
Sandbox Apple ID: Create one at App Store Connect > Users & Access > Sandbox Testers. Do NOT use your real Apple ID for sandbox purchases.
Test: Push notifications Manual
  • Accept notification permission prompt
  • Trigger a notification (e.g., partner activity, or send test via Supabase edge function)
  • Verify notification appears on device
Test: Edge cases Manual
  • Kill app and reopen → session persists, no re-login
  • Toggle airplane mode → app doesn't crash, shows graceful errors
  • Rotate device → app stays portrait (orientation locked)
  • Sign out and sign back in → data loads correctly
  • Test on older device (iPhone SE if possible) → no layout issues
